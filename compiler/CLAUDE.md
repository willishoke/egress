# compiler/

The TypeScript layer. Every file here is a step from a high-detail
surface IR (`.trop` source / `tropical_program_2` JSON) toward a
low-detail emit IR (`tropical_plan_4`, a flat instruction stream over
typed scalar slots). Each step retires some structure the next step
doesn't have to handle. This file walks the pipeline in the order the
structure is dropped.

For the bigger-picture framing — programs as a cartesian category of
typed signal-flow graphs with a guarded trace — see the root
`CLAUDE.md`. The vocabulary doesn't need to be load-bearing day to
day, but the shape of every pass matches it.

```
parse  →  elaborate  →  specialize  →  sumLower  →  traceCycles
       →  inlineInstances  →  arrayLower  →  compileResolved (or interpret_resolved, or emit_wasm)
```

No audio runs in this layer; we produce JSON / WASM bytes and hand them
to the engine.

## Layout

`flatten.ts`, `interpret.ts`, `emit_numeric.ts`, and `ir/load.ts` /
`ir/program_type_builder.ts` are gone. The post-Phase-D path is
`elaborate → strata → compile_resolved` (or `interpret_resolved` /
`emit_wasm`), with `materialize_session` injecting sessions into the
same pipeline.

## Surface in (parse + raise)

`compiler/parse/` is the entry from `.trop` source. Four layers:
`lexer.ts` (tokens), `expressions.ts` (precedence + combinator
lambdas), `statements.ts` (block bodies, decls, assigns),
`declarations.ts` (program signatures, ports, type params).

Output is a `ParsedProgram` (`compiler/parse/nodes.ts`). What this pass
drops:

- whitespace, comments, file layout
- bounds annotations: `lower_bounds.ts` desugars `signal`, `unipolar`,
  `freq`, and explicit `in [lo, hi]` to `clamp` / `select` ops *during
  parse*, before anything else sees the program

What it deliberately *doesn't* do: name resolution. Every reference
emits a `NameRefNode` placeholder. The parser is scope-agnostic by
design — that's the elaborator's job.

`raise.ts` is the legacy JSON-ingest adapter: it walks a
`tropical_program_2` JSON and builds the same `ParsedProgram` shape,
also emitting `NameRefNode`s. The two front-ends (`.trop` parser, JSON
raiser) converge on one input to the elaborator.

## Elaborate (drops names)

`ir/elaborator.ts` is the unique site of name resolution. A single
top-down pass: each declaration is constructed once when its parsed
counterpart is encountered, registered in the appropriate scope, and
re-used by reference at every site that names it.

Output: `ResolvedProgram` (`ir/nodes.ts`). Every reference (`InputRef`,
`RegRef`, `DelayRef`, `ParamRef`, `TypeParamRef`, `BindingRef`,
`NestedOut`) carries a direct decl-object pointer. After this pass, no
string lookups, no scope walks. Decl identity (`===`) is the only
substrate the rest of the compiler operates on.

The IR admits cycles (delays, feedback). The elaborator produces a
graph; later strata are responsible for breaking cycles when needed.

## Strata (the drop sequence)

`ir/strata.ts` composes five passes. Each pass is pure: returns a
fresh `ResolvedProgram` (or, in the no-op fast path, the input by
identity).

| Pass               | What it does |
|--------------------|--------------|
| `specialize`       | Drops type parameters. Substitutes integer values for `TypeParamDecl` refs in shape dims and expression-position `TypeParamRef`. Each `(template, args)` pair produces a fresh `ResolvedProgram`; the cloner shares sum/struct/alias type defs to preserve variant identity. |
| `sumLower`         | Drops sum types. Sum-typed `DelayDecl`s decompose into one tag (int) slot plus one scalar slot per `(variant, field)` pair across all variants. `MatchExpr` becomes a select-chain; `TagExpr` becomes a tag literal + scalar writes. After this pass the IR has no `tag`, `match`, or sum-typed delay. |
| `traceCycles`      | The trace operator's implementation. Tarjan SCC over inter-instance dependencies; each non-trivial SCC gets a synthetic `DelayDecl` on a chosen back-edge (init `0`), and dependent `NestedOut` refs rewrite to read it. The synthetic delay is tagged `_liftedFrom: 'synthetic'`. Causality is enforced by routing every cycle through the unit-delay endomorphism. |
| `inlineInstances`  | Drops nesting. Each `InstanceDecl` is spliced into its parent: inner regs/delays lift into the outer body (renamed `${instance}_${inner}`, tagged `_liftedFrom: ${instance}`); inner inputs substitute the wired-in expressions; `NestedOut` refs replace by the inlined output expressions. After this pass the body has no `InstanceDecl` and no `NestedOut`. |
| `arrayLower`       | Drops shapes and combinators. `let`, `fold`, `scan`, `generate`, `iterate`, `chain`, `map2`, `zipWith` unroll via static shape dims; their `BinderDecl`s are substituted away. `zeros{count}` materializes to inline arrays. `index` and `arraySet` survive as scalar ops over slot bundles. |

Post-strata invariants:

- scalar-only (no surviving array decls; arrays survive only as inline
  literals in expressions or as backing stores for stateful arrays)
- monomorphic (no `TypeParamDecl`, no `TypeParamRef`)
- acyclic (every cycle is broken by a `DelayDecl`)
- non-nested (no `InstanceDecl`, no `NestedOut`)
- combinator-free (no `let`, `fold`, etc.)
- decl-identity-keyed (refs hold decl objects, never strings)

That's the smallest sub-IR that's sufficient for any per-sample
evaluator. The three backends below operate on this image.

## Materialize / compile boundary

Two files split the session-emit responsibility:

- **`ir/materialize_session.ts`** — the session→top-level lift.
  Builds a synthetic `ResolvedProgram` whose body has one
  `InstanceDecl` per session instance, translates session
  `ExprNode`-shaped wiring into `ResolvedExpr`, materializes
  `dac.out` graph_outputs as `OutputAssign`s, synthesizes `ParamDecl`s
  on demand, extracts session-level `delay()` nodes into synthetic
  `DelayDecl`s, and threads gate expressions through the strata
  pipeline (gateable instances get a two-phase wrap — pre-strata on
  the cloned type's outputs and own decls, post-strata on lifted
  sub-instance decls keyed off `_liftedFrom`). Output is a post-strata
  `ResolvedProgram` plus a `Map<string, ParamDecl>` for paramHandle
  stitching. Shared by `compile_session.ts` and
  `interpret_resolved.ts` so the JIT and the oracle see the same IR.

- **`ir/compile_session.ts`** — JIT bookend. Calls
  `materializeSessionForEmit`, builds a `Map<ParamDecl, {ptr}>` from
  the session's param/trigger registries, and hands both to
  `compileResolved`.

The per-program path uses `programTypeFromResolved(prog, typeArgs)`
(`ir/strata.ts`): full strata pipeline, then `new ProgramType(...)`.
The session path uses `compileSession(session)`. Both paths produce
post-strata `ResolvedProgram`s that the backends accept uniformly.

## Backends

Three interpretations of the post-strata IR. None of them is a further
compiler stage — they're parallel emits.

- **`ir/compile_resolved.ts`** → `tropical_plan_4`. Calls
  `buildSlotMaps` (decl identity → integer), then
  `emit_resolved.emitNumericProgram`. The result is a `FlatProgram`
  matching the `tropical_plan_4` schema in `flat_plan.ts`. The C++
  JIT consumes it. `emit_resolved.ts` keys CSE on a bottom-up
  structural id (interned via `${op}|${field=}|${child_id}` strings),
  not node identity, so duplicates introduced by clone-then-substitute
  passes still collapse. Operand kinds: `const`, `input`, `reg`,
  `array_reg`, `state_reg`, `param`, `rate`, `tick`. Array-loop ops
  carry `loop_count > 1` plus `strides` for broadcast vs. iterate.
  Gateable group metadata (when `sourceTag` wraps survive) ships in
  `groups`.

- **`interpret_resolved.ts`** — pure-TS evaluator over `ResolvedExpr`
  against a state map keyed by decl identity. No FFI. The independent
  oracle for `jit_interp_equiv.test.ts`. Reaches every backend that
  rests on the same post-strata IR; a JIT bug shows up here as a
  divergence.

- **`emit_wasm.ts`** + **`wasm_memory_layout.ts`** — emits a
  standalone WASM module with a single exported
  `process(buffer_length, start_sample_index)` function. Linear-memory
  layout (inputs / registers / temps / arrays / param_table /
  param_frame / output, all 8-byte aligned) is shared with
  `web/worklet/runtime.ts` so offsets stay in sync. Runtime details
  in `web/CLAUDE.md`.

## ProgramType / ProgramInstance

`program_types.ts`. Thin wrapper over a post-strata `ResolvedProgram`.
The wrapper is metadata; the IR is the value.

```typescript
class ProgramType {
  readonly prog: ResolvedProgram
  get name(): string
  get inputNames():    string[]
  get outputNames():   string[]
  get inputPortTypes():    (PortType | undefined)[]
  get outputPortTypes():   (PortType | undefined)[]
  get registerNames():     string[]            // lazy via buildSlotMaps
  get registerPortTypes(): (PortType | undefined)[]  // lazy via buildSlotMaps
  get rawInputDefaults():  Record<string, ExprNode>  // lazy
  rename(newName: string): void                // for cache-key rebranding
  instantiateAs(name, opts?): ProgramInstance
}
```

`ProgramInstance` holds a `ProgramType` plus an instance name and
optional `typeArgs`; getter accesses delegate to `type`. There is no
`_def` field, no slot-indexed copy, no upfront flattening. When you
need port metadata, you walk `prog.ports`; when you need register
slots, the lazy cache runs `buildSlotMaps` once.

`session.ts:resolveProgramType` calls `programTypeFromResolved`
(strata + wrap), then `type.rename(key)` so the specialization cache
key (`Type<N=8>`) shows up as the program's name in serialized output.

## Stdlib

31 `.trop` files. See `stdlib/README.md` for the full catalogue and
`compiler/parse/` for the surface syntax. Loaded by `loadStdlib()`
(`compiler/program.ts`) on disk; for browser builds the same files are
inlined into `stdlib_bundled.ts` by `web/bundle_stdlib.ts` and consumed
by `loadStdlibFromSources()`.

## FFI bridge (`runtime/`)

- `bindings.ts` — koffi declarations matching `tropical_c.h`. Loads
  `libtropical.dylib` from `build/`.
- `runtime.ts` — `Runtime` wraps `tropical_runtime_t`. FinalizationRegistry
  for GC-driven cleanup.
- `audio.ts` — `DAC` wraps `tropical_dac_t`. Static `listDevices()`.
- `param.ts` — `Param` (smoothed) and `Trigger` (fire-once). Wiring
  references them by name (`{op:'param', name}` /
  `{op:'trigger', name}`); the materializer resolves the name to a
  handle (`tropical_param_t` for the JIT, SAB slot index stringified
  to `param.ptr` for WASM) at compile time.

## Adding a program type

1. Create `stdlib/MyType.trop` in surface syntax.
2. `loadStdlib()` discovers it on next session boot.
3. Run `bun run scripts/validate_stdlib.ts` to confirm it parses,
   elaborates, and lowers cleanly.
4. No C++ changes unless you need a new expression op (and even then
   the new op needs to land in `WireFormatOp`, the strata passes, and
   the three backends — `emit_resolved`, `interpret_resolved`,
   `emit_wasm` — together).
